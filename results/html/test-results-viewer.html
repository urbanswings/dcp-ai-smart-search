How It Works

  • Runtime mode: choose your tests/json (or results/json) folder and filters load automatically.
  • Embedded mode: if embeddedResultsData has files, results load on page open.
  • This viewer runs fully client-side with no external JS required.
  • Jira Integration: Open a failed query in the selected row view and use Create JIRA there. A preview opens so you can verify/edit the summary and description, then submit directly to Jira via the local bridge (npm run jira:bridge), which authenticates with your JIRA_API_TOKEN. Configure Jira using the ⚙️ icon.

Jira Configuration

Run npm run jira:bridge locally first. It reads JIRA_API_TOKEN from your .env and creates the issue via the Jira REST API.
Jira's "Affected Market(s)" is an option-based multiselect, so codes like JP are matched live against Jira's real option list (e.g. "JP (Japan)") — not sent as raw text.
